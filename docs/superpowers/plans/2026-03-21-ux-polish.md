## Task 2: Escape dismisses all overlays (GraphView + OrphanFinderView in ContentView)

**Files:**
- Modify: `Sources/EzmdvApp/Views/ContentView.swift`

### Context

`ContentView` wraps `GraphView` and `OrphanFinderView` as `.overlay` modifiers. Each overlay `ZStack` already has a background scrim with `.onTapGesture` to dismiss. Escape handling is missing entirely for both.

The `CommandPalette` overlay is dismissed from within `CommandPalette` itself via `.onKeyPress(.escape)`. For the graph and orphan finder, the simplest approach is to add `.onKeyPress(.escape)` directly on the `ZStack` (the full-screen scrim container), since the scrim itself is a `Color` that can receive focus.

However, `Color` does not receive key events. The reliable macOS 14+ SwiftUI pattern is to add `.onKeyPress` to the `ZStack` and make it focusable, or attach it to the overlay's content view.

The cleanest approach that doesn't require adding focus management to `GraphView` is to attach `.onKeyPress(.escape)` to the `ZStack` scrim container in `ContentView` and give it `.focusable()` so it can receive key events, OR to use `NSEvent` monitoring from the overlay modifier. Given that `OrphanFinderView` already will handle Escape in Task 1, and the graph overlay has no focus surface at all, the safest approach for `GraphView` is a `NSEvent` local monitor published via `.onAppear`/`.onDisappear` on the `ZStack`.

For a pure SwiftUI approach (no AppKit bridging): attach `.background(Color.clear.onKeyPress(.escape) { ... })` on the overlay ZStack, plus `.focusable()` on the ZStack and `.defaultFocus(...)`. In practice on macOS 14 SwiftUI, `.onKeyPress` on a `ZStack` with `.focusable()` works reliably when the overlay is the front-most view.

- [ ] **Step 2.1: Add Escape handler to the GraphView overlay ZStack in ContentView**

In `ContentView.swift`, locate the graph overlay block (lines 38–45). The `GraphView` is shown without a scrim — it's positioned with `.overlay(alignment:)`. Since there's no scrim `ZStack`, attach Escape handling directly in `ContentView` by observing `NSEvent` via `onReceive` on a notification, or by wrapping the existing overlay `if showGraph` block in a `ZStack` with a clear background.

The cleanest minimal change: add an `NSEvent` local monitor in `ContentView` that fires only when `showGraph` or `showOrphanFinder` is true. Add these stored properties and lifecycle modifiers.

In `ContentView`, add a stored property for the event monitor:

```swift
@State private var escapeMonitor: Any? = nil
```

Then add `.onAppear` and `.onDisappear` on `ContentView`'s root (after the existing `.animation` modifiers):

```swift
.onAppear {
    escapeMonitor = NSEvent.addLocalMonitorForEvents(matching: .keyDown) { event in
        if event.keyCode == 53 { // Escape
            if showGraph {
                showGraph = false
                return nil // consume the event
            }
            if showOrphanFinder {
                showOrphanFinder = false
                return nil
            }
        }
        return event
    }
}
.onDisappear {
    if let monitor = escapeMonitor {
        NSEvent.removeMonitor(monitor)
        escapeMonitor = nil
    }
}
```

> **Note on key code 53:** This is the hardcoded macOS key code for Escape. It never changes. Using `NSEvent.addLocalMonitorForEvents` is the standard AppKit pattern for catching key events at the window level in SwiftUI apps that don't have a focused text field consuming Escape.

### Context

`EmptyStateView` is currently a standalone struct with no access to `AppState`. It shows a generic icon, "Select a file to view", and a static hint. The spec asks for:

For each recent file path, we need the filename. Use `(path as NSString).lastPathComponent` — no need to look up the full `MarkdownFile` object.

To open a recent file we need both the `projectId` and `filePath`. We must find the project by scanning `appState.projects`. Extract this into a private helper function that mirrors the pattern in `CommandPalette.findFile(at:)`.

- [ ] **Step 4.1: Add `@EnvironmentObject` to `EmptyStateView`**

Replace the current `EmptyStateView` struct (lines 94–109 in `ContentView.swift`) with the new version below. The replacement is a complete rewrite of the struct body only — the struct declaration line stays the same pattern.

```swift
struct EmptyStateView: View {
    @EnvironmentObject var appState: AppState

    var body: some View {
        VStack(spacing: 0) {
            Spacer()

            // Icon and headline
            VStack(spacing: 12) {
                Image(systemName: "doc.richtext")
                    .font(.system(size: 48))
                    .foregroundStyle(.tertiary)
                Text("Select a file to view")
                    .font(.title3)
                    .foregroundStyle(.secondary)
            }

            // Recent files
            let recents = recentItems()
            if !recents.isEmpty {
                VStack(alignment: .leading, spacing: 0) {
                    Text("RECENT")
                        .font(.system(size: 9, weight: .bold))
                        .tracking(1)
                        .foregroundStyle(.tertiary)
                        .padding(.horizontal, 16)
                        .padding(.top, 28)
                        .padding(.bottom, 6)

                    ForEach(recents, id: \.path) { item in
                        Button(action: {
                            appState.openFile(projectId: item.projectId, filePath: item.path)
                        }) {
                            HStack(spacing: 8) {
                                Image(systemName: "clock")
                                    .font(.system(size: 11))
                                    .foregroundStyle(.secondary)
                                VStack(alignment: .leading, spacing: 1) {
                                    Text(item.name)
                                        .font(.system(size: 12, weight: .medium))
                                        .foregroundStyle(.primary)
                                        .lineLimit(1)
                                    Text(item.projectName)
                                        .font(.system(size: 10))
                                        .foregroundStyle(.tertiary)
                                        .lineLimit(1)
                                }
                                Spacer()
                            }
                            .padding(.horizontal, 16)
                            .padding(.vertical, 6)
                            .contentShape(Rectangle())
                        }
                        .buttonStyle(.plain)
                        .background(Color.clear)
                    }
                }
                .frame(maxWidth: 340)
            }

            // Quick actions
            HStack(spacing: 10) {
                Button(action: { appState.showOpenFolderDialog() }) {
                    Label("Open Folder", systemImage: "folder.badge.plus")
                        .font(.system(size: 12))
                }
                .buttonStyle(.bordered)
            }
            .padding(.top, 24)

            Spacer()
        }
        .frame(maxWidth: .infinity, maxHeight: .infinity)
    }

    // MARK: - Helpers

    private struct RecentItem {
        let path: String
        let name: String
        let projectName: String
        let projectId: UUID
    }

    private func recentItems() -> [RecentItem] {
        var results: [RecentItem] = []
        for path in appState.recentFilePaths.prefix(5) {
            guard let (project, file) = findFile(at: path) else { continue }
            results.append(RecentItem(
                path: path,
                name: file.name,
                projectName: project.name,
                projectId: project.id
            ))
        }
        return results
    }

    private func findFile(at filePath: String) -> (Project, MarkdownFile)? {
        for project in appState.projects {
            guard let files = project.files else { continue }
            if let file = MarkdownFile.flatten(files).first(where: { $0.path == filePath }) {
                return (project, file)
            }
        }
        return nil
    }
}
```
